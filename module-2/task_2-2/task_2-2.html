1. Where should you use constants, and where should you use variables? Can they all be constants?

    2. [Use constants (const) for values that never change. Use variables (let) for values that need to change (e.g., counters, user input). Not everything can be a constant because many values are dynamic and require reassignment.]

  2. How do parentheses affect the arithmetic outcome of an expression?

    3. [Parentheses determine the order of operations in an expression. They ensure calculations inside them are performed first, overriding the default precedence of arithmetic operators.]

  3. How do we limit floating-point numbers to a given number of decimals?

    4. [In JavaScript, we can use toFixed(), Math.round(), Intl.NumberFormat, or Math.trunc() with multiplication and division to limit decimals.]

  4. How can we compare two floating-point values to see if they are equal (is 12.199999 the same as 12.2)?

    5. [We can round both numbers to the same decimal place using toFixed() or Math.round() and then comparing them directly]

  5. What do comparison operators return, and how can we check if they are FALSE or TRUE?

    6. [Comparison operators return true or false. You can check the result directly in an if statement or log it, e.g., if (a > b) or Print(a === b)]

  6. How does the Math.random() function work, and what can it be used for?

    7. [Math.random() generates a pseudorandom number between 0 (inclusive) and 1 (exclusive). It is used for tasks like random selections, generating random integers, or simulating randomness in applications.]

  7. What is modulus (%), and when do we need this operator in mathematical operations?

    8. [The modulus operator (%) returns the remainder of a division. It is used for tasks like checking divisibility, cycling through values, or determining even/odd numbers]
